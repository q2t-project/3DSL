## なぜ図なのか。なぜスケールなのか。

この Docs は「仕様書の全量」ではなく、まず迷わず使うための入口です。  
最短ルートは次のとおりです。

1. `/library` を眺め、気になった図を開きます  
2. `/app/viewer` で図を操作し、全体→局所を往復します  
3. 必要に応じて `/canonical` でスキーマと検証方針を確認します  

3DSL は「図を描く道具」というより、「関係を言語化するための枠」を作るプロジェクトです。したがって Docs では、設計の理由（なぜこうしたか）と、運用の前提（どこまでを保証するか）を優先してまとめます。

### 発散と収束

理解や探索は、視野を広げる局面（発散）と、追い込む形を固める局面（収束）を往復します。  
マルチスケール UI は、この往復を道具として支えることを狙っています。

### 引用と表現

3DSL Library に掲載される図は、研究・教育・理解促進を目的として、原著作物を正当に引用し、その関係構造を可視化したものです。図は原著作物の代替閲覧を目的とするものではなく、内容の探索や詳細な参照は出典元またはリンク先（Viewer）に委ねられます。著作権は、それぞれの権利者に帰属します。
