# 作業ルール

本ルールは ChatGPT + 人間 + Codex の三者で作業を進める際の共通規範である。

---

## 三者の役割

- ChatGPT  
  - 仕様・設計・調整を担当  
  - 必ず完全版を提示し、曖昧な差分のみの提示は禁止  

- Codex  
  - コード生成・修正を担当  
  - 必ず差分形式で出力し、完全版を生成する場合は人間確認を経る  

- 人間  
  - CLI / GitHub 操作、デプロイを担当  
  - 判断が必要な場面で意思決定を行う  

---

## 運用ルール

1. **差分管理**  
   - ChatGPT はコードを直接修正せず、Codex に差分適用させる。  

2. **リセット手順**  
   - ルール逸脱が発覚した場合は即リセット宣言。  
   - 直前の仕様書・コードの完全版を ChatGPT が再提示し、作業を再開する。  

3. **保存と同期**  
   - 正典は GitHub リポジトリの内容とする。  
   - ローカル `/3DSL/` とリモート `q2t-project/3DSL` を常に同期する。  

---

## 補則

- Viewer / Modeler / 共用モジュールは必ず 3DSS に準拠する。  
- 補助的な資料（Mermaid, PUMLなど）は最小限に留め、可能なら3DSD図式で代替する。  
- LICENSE, README など公開用文書は support/docs 以下に整理する。
